APPdynamics产品
  • 下一代应用性能管理

    分布式 web 应用通常包含令人沮丧的盲点和难解、反复出现的问题。


    APM 能提供运维和开发团队所需要的简单性、可见性和 深度诊断。


    分布式 web 应用的新面貌对负责确保应用健康和高性能的人员带来了全新的挑战。 现代应用的构架、新技术和快速变化使当今的应用产生了复杂性的完美风暴。 因此，性能问题浮出 表面，而它们通常很难识别、诊断和修复。 当这些应用变得对业务愈加关键时，现在比以往任何时候都更需要一种简单但快速的方法， 用于在应用问题影响收入之前监测、诊断和解决它们。

  • AppDynamics APM 介绍

    应用性能管理解决方案为分布式应用提供以业务事务为中心的管理。 


    该解决方案极其容易配置和部署，只需消耗极少的生产开销，可监测每一行代码，并可确保性能 维持在基线之上，从而在应用性能问题影响客户和业务之前识别和解决它们。






  • 关键优势

    使您的整个应用可视化，从浏览器、移动应用到后端数据库

    Monitor 监测包含 Java、.NET、PHP、Node.js、Python 和 C++(Beta) 在内的混合环境

    通过深度代码级诊断，使瓶颈的故障排除速度提升 90%

    通过 Application Run Book Automation自动修复常见问题

    通过单一虚拟管理面板进行跨层级的监测、基于角色的视图查看和虚拟化作战室，提升态势感知和开发运维团队的协作。

    是什么让 AppDynamics 与众不同？
    – 通过 Transaction Tag and Follow 自动发现和监测端到端业务事务性能
    – 自动发现应用的拓扑结构和依赖关系，包括外部 web 服务，并根据生产应用行为追踪关 键业务事务
    – 为业务事务的性能提供可视化和安排优先级，其中包括应用和基础设施节点的健康 通过业内最广泛的语言和技术了解一切
    – 涵盖所有流行的编程语言和框架，包括 Java、.NE、Node.js、PHP、Python 和 C/C++ (Beta)
    – 涵盖最复杂的企业平台和解决方案，比如 JMS、排队技术、TIBCO、WebMethods 等
    – 利用平台的可扩展性，扩大了应用监测的覆盖范围



    在几分钟内监测任何生成应用
    – 不需要任何配置即可监测任何应用
    – 即使没有源代码也可以监测应用



    以代码级深度监测生产应用且无需开销
    – 利用 Smart Code Instrumentation 深度监测生产应用，无需更改配置
    – 监测每个事务，但智能地只获取异常事务的详情，使平台可扩展以满足大型企业的需求



    快速解决应用问题，带来非凡的最终用户体验
    – 利用 DevOps Virtual War Room 功能使每个人都可以相互协作和排除故障，同时分享相 同的性能数据
    – 通过 Runbook Automation 自动获取快照并在性能问题正在发生时解决它们，从而避免 繁琐的手动步骤。



    通过基线和警报自动设置杜绝假警报
    – 在自动生成动态基线的背景下了解性能
    – 集成事件和警报系统 ServiceNow、PagerDuty 和 Jira 灵活的部署选项：Saas、本地或混合部署
